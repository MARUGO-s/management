## 3. 背景：データ不一致レポートについて

- 現象として、**画面上の「データ不一致」系レポートで登録件数が 0 と表示される**一方で、**実際の Google スプレッドシート（Loan management）には行が追加されている**ケースがあった。  
- 原因の一つは、**クライアント側で比較用に読み取った行データを、報告 API に渡すオブジェクトに正しく載せていなかった**ことによる**表示・集計上の不具合**であった。  
- 今回の修正により、**読み取れた行は `registeredData` として報告データに含まれる**ため、上記の**誤った「0件」表示は解消される想定**である。  
- なお、**送信値とセル値の微妙な差・読み取り範囲・時間窓**などにより、**真の不一致判定**が出る可能性は別途残る（仕様・今後の改善余地）。

## 5. Google Apps Script（本番）について

- リポジトリ内の `.gs` は**サンプル・正本**として管理。  
- **実際の本番動作は、Google の Apps Script エディタに貼り付け・デプロイした内容**が適用される。  
- 本番で No.3 と同等の内容にするには、当該 `.gs` を Google 側に反映し、**Web アプリのデプロイを更新**すること（実施済みである場合は追加作業不要）。

---

## 6. Supabase について

- **今回の修正範囲では、Edge Functions や DB マイグレーションの変更は行っていない。**  
- 既に本番 Supabase に必要な関数・テーブルが存在し、アプリから接続できている場合、**Supabase の再デプロイは必須ではない**。  
- 今後、関数やスキーマを変更した場合は、従来どおり `supabase functions deploy` / `supabase db push` 等が必要。
